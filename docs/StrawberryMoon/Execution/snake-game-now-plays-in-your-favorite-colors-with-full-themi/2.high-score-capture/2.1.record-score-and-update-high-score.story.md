# Record Score and Update High Score

## User Story Statement
As a new user, I want to record each score and overwrite the high score indicator if it beats the previous high score, so that I can track my best performance and see my progress improve over time.

## Acceptance Criteria
- Current session score increments by 1 each time food is consumed with real-time UI update
- Best score is automatically compared after each food consumption event
- If current score exceeds stored best score, best score updates immediately in UI and localStorage
- Best score persists across browser session closes, tab switches, and page refreshes
- Best score persists across browser cache clearing and localStorage.clear() operations
- localStorage key-value pair is created on first game initialization if not present
- Current session score and all-time best score both display in game UI before gameplay begins
- Visual confirmation message or indicator shows when high score has been beaten and saved
- Invalid or malformed stored scores trigger automatic reset to 0 with no UI errors
- localStorage quota exceeded is handled gracefully with user notification or fallback behavior
- Score persistence logic has zero performance impact on game loop or rendering pipeline
